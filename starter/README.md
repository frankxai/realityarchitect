# starter/ — forkable agent templates

One template per move of the Architect's Loop. They're **harness-agnostic** — the role/instructions work in Claude
Code, Cursor, Codex, or any agent runner. Copy a file, drop it into your agent's instructions (or `.claude/agents/`,
`.cursor/rules/`, etc.), and adapt the bracketed `[…]` slots to your context.

Don't install all five. Run the [assessment](https://realityarchitect.ai/assess), find your first gap, fork **that** one.

| File | Move | Build |
|------|------|-------|
| `01-see-memory-vault.md` | See | An intelligence layer your agents query before acting |
| `03-build-first-agent.md` | Build | One agent that reliably does one job you used to do by hand |
| `04-automate-content-loop.md` | Automate | A loop that turns one capture into many finished outputs |

Moves 02 (Design) and 05 (Compound) ship next. Want them first? → https://realityarchitect.ai/assess
